# 커피 주문 앱

## 1. 프로젝트 개요

### 1.1 프로젝트명
커피 주문 앱

### 1.2 프로젝트 목적
사용자가 커피 메뉴룰 주문하고, 관리자가 주문을 관리할 수 있는 간단한 풀스택 웹 앱

### 1.3 개발 범위
- 주문하기 화면(메뉴 선택 및 장바구니 기능)
- 관리자 화면(재고 관리 및 주문 상태 관리)
- 데이터를 생성/조회/수정/삭제할 수 있는 기능

## 2. 기술 스택
- 프론트엔드: HTML, CSS, 리액트, 자바스크립트
- 백엔드: Node.js, Express
- 데이터베이스: PostgreSQL

## 3. 기본 사항
- 프런트엔드와 백엔드를 따로 개발
- 기본적인 웹 기술만 사용
- 학습 목적이므로 사용자 인증이나 결제 기능은 제외
- 메뉴는 커피 메뉴만 있음

## 4. 주문하기 화면 UI (프런트엔드 PRD)

### 4.1 화면 개요
- **화면명**: 주문하기
- **목적**: 사용자가 메뉴를 선택하고 옵션을 적용한 뒤 장바구니에 담아 주문할 수 있게 한다.
- **네비게이션**: 상단에서 '주문하기' | '관리자' 탭으로 화면 전환. 현재 화면은 '주문하기'가 활성 상태로 표시된다.

### 4.2 레이아웃 구조

| 구역 | 위치 | 설명 |
|------|------|------|
| 상단 네비게이션 | 최상단 | 브랜드(COZY), 화면 탭(주문하기 / 관리자) |
| 메뉴 영역 | 중앙 | 메뉴 카드들이 가로로 배치된 영역 |
| 장바구니 영역 | 하단 | 선택한 항목 목록, 합계, 주문하기 버튼 |

### 4.3 상단 네비게이션
- **좌측**: 앱 브랜드명 "COZY" (로고/텍스트, 다크 그린 아웃라인 등으로 구분 가능하게 표시)
- **우측**: 두 개의 탭/버튼
  - **주문하기**: 현재 화면, 활성 상태(더 진한/강조된 스타일)
  - **관리자**: 비활성, 클릭 시 관리자 화면으로 이동

### 4.4 메뉴 카드 (메뉴 영역)
- 메뉴 항목마다 **하나의 카드**로 표시하며, 카드는 가로로 나열(그리드/플렉스).
- 각 카드 구성 요소:
  1. **이미지 영역**: 상품 이미지(없을 경우 placeholder, 예: X 표시된 회색 박스)
  2. **메뉴명**: 굵은 글씨 (예: 아메리카노(ICE), 아메리카노(HOT), 카페라떼)
  3. **가격**: "원" 단위 표기, 천 단위 콤마 (예: 4,000원, 5,000원)
  4. **간단한 설명**: 한 줄 설명용 placeholder 텍스트 (예: "간단한 설명...")
  5. **옵션 (선택 가능)**  
     - **샷 추가 (+500원)**: 체크박스 형태, 선택 시 500원 추가  
     - **시럽 추가 (+0원)**: 체크박스 형태, 선택 시 0원 (가격 변동 없음)
  6. **담기 버튼**: 카드 하단, "담기" 라벨. 클릭 시 선택한 옵션과 함께 해당 메뉴가 장바구니에 추가됨.

### 4.5 장바구니 영역
- **제목**: "장바구니"
- **스타일**: 연한 회색 배경의 둥근 모서리 박스로 구분.
- **표시 내용**  
  - **항목 목록**: 담긴 각 항목에 대해  
    - 메뉴명 + 선택 옵션(있는 경우 괄호로 표기, 예: "(샷 추가)")  
    - 수량: "X 1", "X 2" 형식  
    - 소계 금액 (원, 천 단위 콤마)  
    - 예: `아메리카노(ICE) (샷 추가) X 1 → 4,500원`, `아메리카노(HOT) X 2 → 8,000원`
  - **총 금액**: "총 금액 12,500원" 형식, 굵게 강조.
  - **주문하기 버튼**: 장바구니 박스 하단, 우측 정렬. "주문하기" 라벨. 클릭 시 주문 확정/전송(결제는 제외).

### 4.6 사용자 인터랙션 정리
- **옵션 선택**: 메뉴 카드에서 "샷 추가", "시럽 추가" 체크 후 "담기" 클릭 시 해당 옵션이 적용된 상태로 장바구니에 추가.
- **담기**: 메뉴 카드의 "담기" 클릭 → 동일 메뉴+옵션 조합이 이미 있으면 수량 증가, 없으면 새 줄로 추가. 소계/총 금액 자동 계산.
- **수량**: 장바구니에서 항목별 수량 표시(예: X 1, X 2). 수량 변경/삭제 UI(예: +/- 버튼)는 와이어프레임에 명시되지 않았으나, 구현 시 수량 변경 및 항목 삭제 기능을 두는 것을 권장.
- **주문하기**: 장바구니 하단 "주문하기" 버튼 클릭 → 현재 장바구니 내용으로 주문 전송(백엔드 연동). 학습 범위에 따라 확인 단계(모달/다음 화면) 여부는 선택.

### 4.7 금액 계산 규칙
- **기본가 + 옵션가**: 메뉴 기본가에 "샷 추가" 등 옵션 금액을 더한 값이 항목 단가.
- **소계**: (단가 × 수량)을 해당 항목에 표시.
- **총 금액**: 장바구니 내 모든 항목 소계의 합계.

### 4.8 참고 (와이어프레임 기준)
- 예시 메뉴: 아메리카노(ICE) 4,000원, 아메리카노(HOT) 4,000원, 카페라떼 5,000원.
- 예시 옵션: 샷 추가 +500원, 시럽 추가 +0원.
- 예시 장바구니: 아메리카노(ICE) 샷 추가 1개(4,500원) + 아메리카노(HOT) 2개(8,000원) → 총 12,500원.

## 5. 관리자 화면 UI (프런트엔드 PRD)

### 5.1 화면 개요
- **화면명**: 관리자
- **목적**: 주문 통계를 한눈에 보고, 메뉴별 재고를 조정하며, 들어온 주문을 접수·상태 관리할 수 있게 한다.
- **네비게이션**: 상단에서 '주문하기' | '관리자' 탭으로 화면 전환. 현재 화면은 '관리자'가 활성 상태(더 진한 테두리 등)로 표시된다.

### 5.2 레이아웃 구조

| 구역 | 위치 | 설명 |
|------|------|------|
| 상단 네비게이션 | 최상단 | 브랜드(COZY), 화면 탭(주문하기 / 관리자) |
| 관리자 대시보드 | 상단 아래 | 주문 통계 요약 |
| 재고 현황 | 중앙 | 메뉴별 재고 수량 및 증감 버튼 |
| 주문 현황 | 하단 | 대기 중인 주문 목록 및 액션 버튼 |

### 5.3 상단 네비게이션
- **좌측**: 앱 브랜드명 "COZY" (주문하기 화면과 동일 스타일 유지)
- **우측**: 두 개의 탭/버튼
  - **주문하기**: 비활성, 클릭 시 주문하기 화면으로 이동
  - **관리자**: 현재 화면, 활성 상태(더 진한 테두리/강조 스타일)

### 5.4 관리자 대시보드
- **제목**: "관리자 대시보드"
- **스타일**: 연한 회색 배경의 둥근 모서리 박스.
- **표시 항목** (주문 상태별 건수 요약):
  - **총 주문**: 전체 주문 수 (예: 총 주문 1)
  - **주문 접수**: 접수 완료된 주문 수 (예: 주문 접수 1)
  - **제조 중**: 제조 진행 중인 주문 수 (예: 제조 중 0)
  - **제조 완료**: 제조 완료된 주문 수 (예: 제조 완료 0)
- 대시보드는 현재 운영 상태를 한눈에 파악할 수 있도록 숫자만 간단히 노출한다.

### 5.5 재고 현황
- **제목**: "재고 현황"
- **스타일**: 연한 회색 배경의 둥근 모서리 박스. 메뉴별로 작은 테두리 박스로 구분하여 배치.
- **메뉴별 표시 내용**:
  - **메뉴명**: 예) 아메리카노(ICE), 아메리카노(HOT), 카페라떼
  - **현재 재고 수량**: "N개" 형식 (예: 10개)
  - **재고 조정 버튼**:
    - **"+" 버튼**: 클릭 시 해당 메뉴 재고 1 증가 (백엔드 반영)
    - **"-" 버튼**: 클릭 시 해당 메뉴 재고 1 감소 (백엔드 반영, 0 미만은 방어 로직 고려)
- 주문하기 화면의 메뉴 목록과 동일한 메뉴 단위로 재고를 관리한다.

### 5.6 주문 현황
- **제목**: "주문 현황"
- **스타일**: 연한 회색 배경의 둥근 모서리 박스.
- **표시 내용**: 대기 중인 주문 목록. 각 주문 항목마다 다음 정보 표시:
  - **주문 일시**: 예) "7월 31일 13:00" (주문 접수/생성 시각)
  - **주문 메뉴 및 수량**: 예) "아메리카노(ICE) x 1" (옵션이 있으면 괄호로 표기 가능)
  - **주문 금액**: "N원" 형식, 천 단위 콤마 (예: 4,000원)
  - **액션 버튼**: "주문 접수" — 클릭 시 해당 주문을 접수 처리(상태를 '주문 접수' 등으로 변경). 접수 후 제조 중·제조 완료 등 다음 단계로 진행하는 UI는 와이어프레임 범위 외이나, 구현 시 단계별 버튼 또는 상태 변경을 권장.

### 5.7 사용자 인터랙션 정리
- **탭 전환**: "주문하기" 클릭 → 주문하기 화면으로 이동.
- **재고 조정**: 재고 현황에서 메뉴별 "+" / "-" 클릭 → 해당 메뉴 재고 증감 후 백엔드에 반영. 화면 숫자 즉시 갱신.
- **주문 접수**: 주문 현황 목록에서 "주문 접수" 클릭 → 해당 주문이 접수 상태로 변경. 대시보드의 "주문 접수" 등 해당 상태 건수 갱신. 목록에서 해당 주문의 표시 방식 변경(이동/숨김/상태 라벨 등)은 구현 시 정의.

### 5.8 참고 (와이어프레임 기준)
- 대시보드 예시: 총 주문 1, 주문 접수 1, 제조 중 0, 제조 완료 0.
- 재고 예시: 아메리카노(ICE) 10개, 아메리카노(HOT) 10개, 카페라떼 10개 — 각각 +/- 버튼으로 조정.
- 주문 현황 예시: 7월 31일 13:00, 아메리카노(ICE) x 1, 4,000원, [주문 접수] 버튼.
